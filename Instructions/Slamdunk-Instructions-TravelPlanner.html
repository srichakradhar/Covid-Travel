Slamdunk - TravelPro - Problem Statement

Front-End

  • The challenge is meant to design and develop a travel application that should be able to display basic web page like Home , About us, Contact us, Travel Advisory etc.
  • Create a folder inside the project folder and build your app in any front-end framework with the required functionalities mentioned below.
  • Your front-end app should run on port 8000.

Note:
While submitting the application, your app should run on port 8000. It is required for testing. Otherwise, your submission will not be validated.

Working of the Application

This web application contains 5 pages - Home page, Covid Guidelines page, Travel Guide page, About US page and Contact US page.

Backend Info

  • This application uses NodeJS as the back-end which resides in NodeJS/ folder.(Don't modify the back-end)
  • By default, the back-end has state details and guidelines.
  • There are some default data stored in the database which resides in nodejs/src/mongoose/db/defaultDB.js.
  • Run npm start command from nodejs directory to run the back-end (It should be run on port 8001)

MongoDB Commands

  • You can open the mongo shell by running mongo from the terminal.
  • You can view all the data from the database in MongoDB by running show dbs from the mongo shell.
  • You can select the database by running use states.
  • You can view the names of the collections by running show collections.
  • You can view the data inside a collection by running db.collection_name.find().
  • Enter Ctrl+C to exit.

Covid Guidance Page

  • In this page, user can view the travel guidelines, according to different states.
  • When the page loads, the "api/states/Header?search=guidelines" should fetch the header details from back-end.
  • When the page loads, the "api/states/General?search=guidelines" should fetch the guidelines details from back-end.
  • When the page loads, the "api/states/Operators?search=guidelines" should fetch the operators details from back-end.
  • On clicking each state, the state-wise guidelines should be displayed. "/api/states/${name_of_state}?search=guidelines" should fetch the state guidelines details from back-end.

    • Sample Request 
    /api/states/${TamilNadu}?search=guidelines
  • Use only the state names which is given in the template.
